# 批量处理模式

> 当供应商有多个产品需要同时生成内容时，使用批处理模式提高效率。

---

## 使用场景

- 一个供应商同时有 3-5 个应季产品
- 需要为同一品类下的多个品种生成内容（如各种水果）
- 周期性批量更新内容（每周为在售产品生成新一批视频）

## 批处理流程

```
产品列表 (JSON数组)
    │
    ▼
预处理: 品类分组 + 路线预判 (inline)
    │
    ▼
Phase 2: 市场调研 ── 共享一次调研结果 (同品类/同时段)
    │
    ▼
Phase 3-5: 每个产品并行 ── 独立生成脚本+视觉+标签
    │
    ▼
Phase 6: 统一合成 ── 一份 DOCX 含所有产品
```

## 输入格式

```json
{
  "batch_name": "2026年7月应季水果",
  "supplier": "宾川阳光果园",
  "products": [
    {
      "product_name": "宾川沃柑",
      "category": "水果类",
      "...": "..."
    },
    {
      "product_name": "元谋葡萄",
      "category": "水果类",
      "...": "..."
    },
    {
      "product_name": "蒙自石榴",
      "category": "水果类",
      "...": "..."
    }
  ]
}
```

## 优化策略

### 市场调研共享
同品类、同时段的产品可以共享一次市场调研结果：
- **同品类** → 共享竞品分析和趋势报告
- **同产地** → 共享产地故事和地域标签
- **不同品类** → 各自独立调研

### IP角色复用
同品类的产品使用同一个IP角色，只需切换产品名和卖点：

| 批次 | 产品数 | IP角色 | 新增工作量 |
|---|---|---|---|
| 3种水果 | 3 | 宾川阿伯 ×1 | 切换产品名+卖点，每个产品约30%的定制化 |
| 5种蔬菜 | 5 | 昆阳大嫂 ×1 | 同上 |

### 优先级排序
如果资源有限，按以下优先级处理：
1. **高毛利产品** > 低毛利产品
2. **应季爆款** > 全年常销
3. **有现货** > 预售

## 输出格式

批处理输出为一份合并 DOCX：

```
封面: 批次名称 + 供应商 + 日期

每个产品:
  产品名称（1页分隔）
  → 产品档案
  → 各平台脚本（精简版，3个平台放2页内）
  → 视觉指导要点
  → 标签策略

末尾:
  批次发布日历（跨产品统一排期）
  资源共享清单（同批产品可复用的素材）
```
